# BERT Fine-Tuning for Twitter Multiclass Hate Speech Classification
## Using LoRA on the THOS Dataset



## Project Overview

This project fine-tunes BERT for **3-class hate speech classification** on Twitter using the THOS dataset:
- **Normal** (0)
- **Offensive** (1)
- **Hate Speech** (2)

We address four core problems with vanilla BERT fine-tuning:
1. **Domain mismatch** — Twitter slang vs. formal Wikipedia text
2. **Class imbalance** — Normal >> Offensive >> Hate
3. **Overfitting** on small data — solved with LoRA
4. **Model fragility** — solved with consistent evaluation + seeded runs

### STEP 3 — LoRA Fine-Tuning

```bash
python lora_bert.py
```

**What is LoRA?**

LoRA (Low-Rank Adaptation) injects small trainable matrices into frozen BERT attention layers:
```
W_new = W_frozen + B · A
        where rank(B·A) = r << full rank
```

- Only matrices A and B (and the classifier head) are trained
- BERT's 110M weights remain frozen
- Reduces trainable parameters from ~110M → ~300K (0.27%)
- Prevents overfitting on small datasets

**LoRA hyperparameters:**
| Parameter | Value | Effect |
|---|---|---|
| `lora_r` | 8 | Rank of decomposition (higher = more capacity) |
| `lora_alpha` | 16 | Scaling (alpha/r = effective learning rate scale) |
| `lora_dropout` | 0.1 | Regularisation within LoRA layers |
| `target_modules` | `["query", "value"]` | BERT attention projections to adapt |
| `learning_rate` | 3e-4 | LoRA can use higher LR than full fine-tuning |

**Expected console output:**
```
trainable params: 296,451 || all params: 109,780,995 || trainable%: 0.27
```

**Expected outputs:**
- `outputs/lora_bert/best_model/` — PEFT adapter files only (small!)
- `outputs/lora_bert/training_history.png`
- `outputs/lora_bert/results.json`

### STEP 4 — Focal Loss (Class Imbalance Fix)

```bash
python focal_loss.py
```

**Why Focal Loss?**

Standard CrossEntropy treats a correctly-classified normal tweet and a misclassified hate tweet equally. Focal Loss down-weights easy examples:

```
FL(p_t) = -alpha_t · (1 - p_t)^γ · log(p_t)
```

- `(1 - p_t)^γ` → reduces loss for high-confidence correct predictions
- Minority class (hate speech) has low p_t → gets higher attention
- `alpha_t` → per-class weighting (same as class_weights from Step 1)
- `γ = 2.0` is the standard default from the original paper

**This is the same LoRA architecture as Step 3**, only the loss function changes.

**Import in other scripts:**
```python
from step4_focal_loss import FocalLoss
loss_fn = FocalLoss(gamma=2.0, alpha=class_weight_tensor)
```

**Expected improvement:** Hate speech F1 should increase significantly vs. Steps 2 & 3.

## Design Decisions & Justifications

### Why `bert-base-uncased` and not BERTweet?
We use `bert-base-uncased` as the baseline (per the project spec referencing Devlin et al. 2019). BERTweet is listed as a supporting reference. If you want to test BERTweet, change `model_name` in CONFIG to `vinai/bertweet-base`.

### Why target `["query", "value"]` in LoRA?
Query and value projections contain the most task-relevant information in attention. Targeting key and output matrices too yields diminishing returns for 2x the adapter parameters.

### Why `gamma=2.0` for Focal Loss?
Lin et al. (2017) empirically found gamma=2 works best across multiple imbalanced detection tasks. You can tune it — try gamma in `{0.5, 1.0, 2.0, 5.0}`.

### Why stratified splits?
With only ~10K samples and severe class imbalance (10% hate), random splits can accidentally remove most hate examples from the training set.
